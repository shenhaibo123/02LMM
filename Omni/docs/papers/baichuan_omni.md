| 项目 | 详情 |

| **机构** | 百川智能 |

## 2. 模型架构

### 2.1 视觉编码器

| 项目 | 详情 |
|------|------|
| **名称** | SigLIP-so400m-patch14-384 |
| **参数量** | ~428M |
| **输入分辨率** | 384×384（支持 AnyRes 动态分辨率） |
| **输出** | 729 个视觉 token，经 2×2 Mean Pool 降至 182 个 |

消融实验（Table 8）对比了 7 个视觉编码器（含 CLIP、DFN、EVA、InternViT-6B 等），SigLIP-so400m 以平均 43.80 分胜出，在 OCR 任务上表现尤为突出。值得注意的是，InternViT-6B（5.9B 参数）得分最低（29.99），说明编码器参数量与性能无直接正相关。

### 2.2 音频编码器

| 项目 | 详情 |
|------|------|
| **名称** | Whisper-large-v3 |
| **参数量** | ~1.5B |
| **输入** | 30 秒音频，128 mel 频谱 |
| **输出** | 1280 维音频表示 |

### 2.3 LLM 骨干

| 项目 | 详情 |
|------|------|
| **名称** | Baichuan 系列 7B LLM |
| **参数量** | ~7B |

### 2.4 投影层/适配器

| 模块 | 结构 | 说明 |
|------|------|------|
| **视觉投影器** | 2×2 Conv Mean Pool + 2-layer MLP | 将视觉 token 从 729 降至 182，再投影至 LLM 嵌入维度 |
| **音频投影器（Conv-GMLP）** | 卷积门控 MLP + 残差连接 | 上分支：两层卷积逐级下采样；下分支：卷积+SiLU 门控；两分支相乘后加残差。下采样率可达 8 倍（token 数减至 1/8），ASR 性能仅下降 0.3% |
| **视频投影器** | 2×2 卷积 | 对视觉编码帧做 token 长度调节（182-546 token） |

### 2.5 语音解码器

本模型不包含语音解码器，输出为文本。

## 3. 训练阶段详述

### 阶段一：多模态对齐预训练（Multi-modal Alignment Pre-training）

分为四个子阶段，按分支渐进式对齐。

#### 子阶段 1-1：Image-Language Stage I

| 项目 | 详情 |
|------|------|
| **目标** | 图文初始对齐，让视觉投影器学习将视觉特征映射到 LLM 嵌入空间 |
| **冻结模块** | LLM（7B）、视觉编码器 SigLIP（428M） |
| **训练模块** | 视觉投影器（Mean Pool + 2-layer MLP） |
| **损失函数** | Next-token prediction（自回归语言建模损失） |
| **学习率** | 1e-3 |
| **数据类型** | 图文对齐数据（caption、描述） |

#### 子阶段 1-2：Image-Language Stage II

| 项目 | 详情 |
|------|------|
| **目标** | 加强视觉理解，特别是 OCR 与图表理解能力 |
| **冻结模块** | LLM（7B） |
| **训练模块** | 视觉投影器 + 视觉编码器 SigLIP |
| **损失函数** | Next-token prediction |
| **学习率** | 1e-5 |
| **数据类型** | 图文对齐数据 + OCR/Chart 专项数据 130K |

#### 子阶段 1-3：Image-Language Stage III

| 项目 | 详情 |
|------|------|
| **目标** | 全参数联合训练，进一步提升图文理解与推理；加入纯文本数据维持 LLM 原始语言能力 |
| **冻结模块** | 无（全部解冻） |
| **训练模块** | LLM（7B）+ 视觉编码器 + 视觉投影器 |
| **损失函数** | Next-token prediction |
| **学习率** | 1e-5 |
| **数据类型** | 图文数据 + 交错图文数据（interleaved）+ 纯文本数据（防止 LLM 能力退化） |

#### 子阶段 1-4：Video-Language

| 项目 | 详情 |
|------|------|
| **目标** | 视频理解能力对齐，复用图文训练成果引导视频训练 |
| **冻结模块** | 视觉编码器 SigLIP + LLM（7B） |
| **训练模块** | 视频投影器（2×2 卷积） |
| **损失函数** | Next-token prediction |
| **学习率** | 4e-6 |
| **数据类型** | 先图文数据，再混合视频文数据（渐进式）。视频 1fps 采样，最多 48 帧 |

#### 子阶段 1-5：Audio-Language

| 项目 | 详情 |
|------|------|
| **目标** | 音频-语言对齐，使模型能理解语音与音频内容 |
| **冻结模块** | LLM（7B） |
| **训练模块** | 音频编码器 Whisper-large-v3 + Conv-GMLP 音频投影器 |
| **损失函数** | Next-token prediction |
| **数据类型** | 长音频-文本序列（最长 4K token），包括 ASR 数据、音频描述等 |

#### 子阶段 1-6：Omni-Alignment

| 项目 | 详情 |
|------|------|
| **目标** | 全模态统一对齐，建立跨模态理解基础 |
| **冻结模块** | 无（全部可训练） |
| **训练模块** | 所有模块联合训练 |
| **损失函数** | Next-token prediction |
| **数据类型** | 混合高质量图文、视频文、音频文对齐数据 |

### 阶段二：多模态监督微调（Multimodal SFT）

| 项目 | 详情 |
|------|------|
| **目标** | 跨模态多任务微调，提升理解、推理与交互能力 |
| **训练模块** | 全模型参数 |
| **损失函数** | Next-token prediction（SFT） |
| **数据量** | 600K 实例，200+ 任务类型 |
| **数据类型** | 文本、音频、图文、视频文、图像+音频交互、视频+音频交互 |
| **数据构造** | 跨模态交互数据：文本按 1:3 拆分，前 1/4 用 TTS（44 种音色）转语音作为音频输入，后 3/4 为预测目标 |
| **训练技巧** | packing + flash-attention2 cuseg_len 做样本隔离加速 |
| **数据筛选** | 按模型已有能力筛选数据子集，避免强行注入未知知识导致幻觉 |

### 公开数据集

- **图文**：通用图文对齐数据（未明确列出具体数据集名）
- **OCR/Chart**：130K 专项数据
- **视频**：视频文对齐数据
- **音频**：WenetSpeech、Fleurs（zh/en）、KeSpeech（方言 ASR）、Covost-2（翻译）、AIR-Bench 等
- **评测基准**：MMLU、CMMLU、AGIEval、C-Eval、MMBench、SEED-IMG、MME、MMMU、Video-MME、MVBench 等

## 5. 关键创新

1. **Conv-GMLP 音频投影器**：用卷积门控 MLP 替代传统 stride pooling 做音频下采样，在高下采样率（8 倍）下 ASR 性能仅下降 0.3%，序列压缩鲁棒性极强。
2. **渐进式分支训练策略**：从单模态（图文→视频→音频）逐分支对齐，再做全模态联合对齐（Omni-Alignment），最后多任务 SFT。每阶段目标单一，避免优化冲突。
3. **跨模态交互数据构造**：将文本按 1:3 拆分，前 1/4 用 TTS（44 种音色）转语音作音频输入，创造性地解决了跨模态交互数据稀缺问题。
4. **AnyRes 动态分辨率**：文档理解类任务（DocVQA +14.87 pp、InfographicVQA +15.26 pp）提升极大。
5. **流式音视频交互机制**：模型预测音频输入起止边界，期间流式编码视频帧，音频结束后将音频特征送入推理，实现实时输入处理。

## 6. 启示与借鉴

1. **渐进式训练是关键**：Baichuan-Omni 的分支渐进训练策略证明了「先单模态对齐、再全模态统一、最后多任务微调」的有效性。我们构建 Omni 模型时，应避免一步到位训练多模态，而是分阶段、分模态逐步引入。

2. **音频投影器设计值得借鉴**：Conv-GMLP 在高压缩率下保持音频质量的特性，对于我们处理长音频输入（减少 token 数、降低显存开销）非常有参考价值。

3. **跨模态数据构造方法可复用**：TTS 构造跨模态交互数据的思路简单有效，可直接应用于我们的数据构建流程。多音色 TTS 可增加音频多样性。

4. **数据质量筛选策略**：SFT 阶段按模型已有能力筛选数据，避免注入幻觉，这一策略在我们的微调阶段应予以采纳。

5. **纯文本数据混合防退化**：在图文全参训练阶段混入纯文本数据维持 LLM 原始能力，这是多模态训练中防止语言能力退化的有效手段。

6. **7B 规模的可行性**：Baichuan-Omni 证明 7B 规模足以实现全模态理解与交互，在中文基准上甚至超越 12B 激活参数的 VITA（MoE），为我们选择模型规模提供了参考。
