# OpenOmni 详细训练报告

---

## 1. 模型概览

| 项目 | 详情 |
|------|------|
| **名称** | OpenOmni |
| **机构** | 通义实验室（Tongyi-ConvAI）等 |
| **发布时间** | 2025-01（arXiv:2501.04561），NIPS 2025 接收 |
| **参数量** | ~7B（LLM 骨干 Qwen2-7B 或 Llama3-8B） |
| **一句话总结** | 唯一完整开源训练脚本的全模态 LLM，通过 5 阶段渐进训练（Speech2Text → Image2Text Pretrain → Image2Text Finetune → Text2Speech Pretrain → Text2Speech DPO），在 8×A100-80G 上实现语音理解、视觉理解与情感语音合成的统一，核心创新包括 Emotional DPO 和语言桥接策略。 |

## 2. 模型架构

### 2.1 视觉编码器

| 项目 | 详情 |
|------|------|
| **名称** | 基于 LLaVA 框架的视觉编码器（参考 MMEvol 数据管线） |
| **说明** | 用于图像/视频理解，处理图像帧并输出视觉特征序列 |

### 2.2 音频编码器

| 项目 | 详情 |
|------|------|
| **名称** | 语音离散化编码器 |
| **支持模式** | CTC 模式（6K 词汇表，CosyVoice tokenizer，速度快）和 AR 模式（16K 词汇表，GLM4Voice tokenizer，质量高） |
| **说明** | 将语音波形编码为离散 token，支持双模式切换 |

### 2.3 LLM 骨干

| 项目 | 详情 |
|------|------|
| **名称** | Qwen2-7B / Llama3-8B（支持多骨干切换） |
| **参数量** | ~7B / ~8B |
| **说明** | 作为语言桥梁，统一语音理解、视觉理解和语音生成 |

### 2.4 投影层/适配器

| 模块 | 说明 |
|------|------|
| **语音-文本投影层** | 将语音编码器输出映射到 LLM 嵌入空间 |
| **视觉-文本投影层** | 将视觉编码器输出映射到 LLM 嵌入空间（参考 LLaVA 架构） |
| **文本-语音投影层** | 将 LLM 隐藏状态映射到语音 token 空间 |

### 2.5 语音解码器

| 项目 | 详情 |
|------|------|
| **CTC 模式** | 基于 CosyVoice tokenizer（6K 词汇），推理速度快 |
| **AR 模式** | 基于 GLM4Voice tokenizer（16K 词汇），生成质量更高 |
| **说明** | 双模式设计允许在速度与质量之间灵活选择 |

## 3. 训练阶段详述

OpenOmni 采用 5 阶段渐进式训练，核心思想是「语言桥接」——以文本作为不同模态之间的桥梁，在数据和显存资源稀缺的场景下实现高效全模态训练。

### 阶段 1-1：Speech-to-Text Pretrain（语音到文本预训练）

| 项目 | 详情 |
|------|------|
| **目标** | 使 LLM 具备语音理解能力，建立语音→文本的映射 |
| **冻结模块** | LLM 骨干（部分或全部） |
| **训练模块** | 语音编码器 + 语音-文本投影层 |
| **损失函数** | Next-token prediction（语音输入→文本输出） |
| **数据来源** | AISHELL-4（中文会议语音）、LibriSpeech（英文朗读语音）、WeNetSpeech（中文多领域语音） |
| **数据类型** | 语音-文本配对（ASR 数据） |

### 阶段 2-1：Image-to-Text Pretrain（图像到文本预训练）

| 项目 | 详情 |
|------|------|
| **目标** | 使 LLM 具备视觉理解能力，建立图像→文本的映射 |
| **冻结模块** | LLM 骨干（部分或全部）、语音编码器（保持阶段 1 成果） |
| **训练模块** | 视觉编码器 + 视觉-文本投影层 |
| **损失函数** | Next-token prediction（图像输入→文本输出） |
| **数据来源** | LLaVA-595K（图文对齐数据） |
| **数据量** | ~595K 图文对 |

### 阶段 2-2：Image-to-Text Finetune（图像到文本指令微调）

| 项目 | 详情 |
|------|------|
| **目标** | 提升视觉理解的指令跟随能力和复杂推理能力 |
| **冻结模块** | 视觉编码器 |
| **训练模块** | LLM 骨干 + 视觉-文本投影层 |
| **损失函数** | Next-token prediction（SFT） |
| **数据来源** | MMEvol（多模态进化数据集，含 AI2D 等） |
| **数据类型** | 视觉指令微调数据（问答、推理、描述等） |

### 阶段 3-1：Text-to-Speech Pretrain（文本到语音预训练）

| 项目 | 详情 |
|------|------|
| **目标** | 使 LLM 具备语音生成能力，建立文本→语音 token 的映射 |
| **冻结模块** | 视觉编码器、语音编码器 |
| **训练模块** | LLM 骨干 + 文本-语音投影层 + 语音解码头 |
| **损失函数** | Next-token prediction（文本输入→语音 token 输出） |
| **数据来源** | 合成语音语料（问题和回答的 TTS 合成语音） |
| **数据类型** | 文本-语音 token 配对 |

### 阶段 3-2：Text-to-Speech Emotional DPO（情感语音 DPO）

| 项目 | 详情 |
|------|------|
| **目标** | 提升生成语音的情感表达质量，使语音更自然、更富有情感 |
| **训练模块** | LLM 骨干 + 语音解码相关参数 |
| **损失函数** | DPO（Direct Preference Optimization）偏好学习损失 |
| **数据来源** | ~9K 情感偏好/拒绝对（emotional preference/reject pairs） |
| **数据类型** | (正例: 情感表达好的语音, 负例: 情感表达差的语音) 配对 |
| **说明** | 核心创新——通过 DPO 显式优化语音的情感质量，而非仅优化清晰度或准确度 |

## 5. 关键创新

1. **Emotional DPO（情感直接偏好优化）**：首次将 DPO 应用于语音情感质量优化。通过 ~9K 情感偏好/拒绝对训练，使生成语音在情感表达（而非仅清晰度）上显著提升，解决了 TTS 系统"正确但机械"的痼疾。

2. **5 阶段渐进训练策略**：将全模态训练拆分为 Speech2Text → Image2Text Pretrain → Image2Text Finetune → Text2Speech Pretrain → Text2Speech DPO 五个阶段，每阶段引入一种能力，避免多任务同时训练的优化冲突。

3. **语言桥接（Language Bridging）**：以文本作为不同模态之间的桥梁——语音先转文本理解、图像先转文本理解、文本再转语音输出。这种设计使得各阶段可以独立使用单模态数据训练，大幅降低对全模态配对数据的依赖。

4. **双模式语音生成**：CTC 模式（6K 词汇，快速）和 AR 模式（16K 词汇，高质量）的双模式设计，允许在推理速度和生成质量之间灵活切换，适配不同应用场景。

5. **资源高效训练**：仅 8×A100-80G 即可完成全流程 5 阶段训练，是同类全模态模型中资源需求最低的方案之一，降低了学术和中小团队的复现门槛。

## 6. 启示与借鉴

1. **最佳复现起点**：OpenOmni 是 5 篇论文中唯一完整开源训练脚本的工作，且全部使用公开数据集、硬件需求明确（8×A100-80G）。我们构建 Omni 模型的第一步应该是复现 OpenOmni 的训练流程，建立基线。

2. **5 阶段渐进训练的实操价值**：每个阶段的输入输出、数据来源、冻结/训练模块都非常清晰，可以直接作为我们训练流程的蓝本。特别是「先理解后生成」的顺序（先 Speech2Text 和 Image2Text，后 Text2Speech）符合认知直觉，也更容易调试。

3. **Emotional DPO 的创新可扩展**：DPO 不仅可用于情感语音，还可扩展到语音的其他维度（如语速、音色、口音适应等）。我们在语音输出阶段可以借鉴此思路，构建多维度的偏好数据进行 DPO 训练。

4. **语言桥接降低数据需求**：以文本为中心的桥接策略意味着我们可以分别收集语音-文本数据和图像-文本数据进行训练，无需大量三模态配对数据（语音+图像+文本）。这对数据资源有限的团队尤为重要。

5. **双模式语音生成的灵活性**：CTC 和 AR 双模式设计值得采纳——在实时交互场景使用 CTC 模式追求低延迟，在离线生成场景使用 AR 模式追求高质量。

6. **公开数据集的全覆盖**：OpenOmni 使用的数据集（AISHELL-4、LibriSpeech、WeNetSpeech、LLaVA-595K、MMEvol）全部公开可用，我们可以直接复用这些数据源，无需额外采集或标注。
