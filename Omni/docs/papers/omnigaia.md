## 1. 模型概览

| 项目 | 详情 |
|------|------|
| **名称** | OmniGAIA（基准）+ OmniAtlas（智能体模型） |
| **机构** | 中国人民大学、小红书、东南大学、浙江大学、清华大学 |

## 2. 模型架构

OmniAtlas 是在现有全模态 LLM（如 Qwen3-Omni）基础上进行智能体能力增强的训练方案，而非全新的模型架构。

### 2.1 视觉编码器

继承 Qwen3-Omni 的视觉编码器，支持图像和视频帧处理。

### 2.2 音频编码器

继承 Qwen3-Omni 的音频编码器，支持语音和环境声理解。

### 2.3 LLM 骨干

| 项目 | 详情 |
|------|------|
| **名称** | Qwen3-Omni-30B / Qwen3-Omni-7B |
| **参数量** | 30B / 7B |
| **特点** | 原生全模态模型，支持视觉+音频+文本统一理解 |

### 2.4 工具接口（OmniAtlas 特有）

| 工具 | 功能 |
|------|------|
| **web_search** | 网页搜索，获取外部知识 |
| **page_browser** | 网页浏览，读取页面内容 |
| **code_executor** | 代码执行，完成算术和统计计算 |
| **主动感知工具** | 按需请求特定时间段的视频帧或音频片段，避免整段下采样 |

### 2.5 语音解码器

继承 Qwen3-Omni 的语音输出能力。

## 3. 训练阶段详述

### 阶段一：轨迹合成（Trajectory Synthesis）

| 项目 | 详情 |
|------|------|
| **目标** | 自动合成高质量的「推理+工具调用」训练轨迹，作为 SFT 的正例数据 |
| **方法** | 后见之明引导的树搜索（Hindsight-guided Tree Exploration） |
| **探索代理** | DeepSeek-V3.2 |
| **监督模型** | Gemini-3（对每一步做质量评估） |

**具体流程**：
1. 给定题目和媒体，DeepSeek-V3.2 在每一步生成多个候选推理+工具调用动作，形成探索树
2. 利用已知的正确答案进行「后见之明打分」——判断哪些分支能通向正确答案
3. 从树中选出走到正确答案的完整路径作为正例轨迹
4. Gemini-3 对每一步的合理性做监督，过滤中间步骤含糊或错误的轨迹
5. 保留失败轨迹用于后续 OmniDPO 训练

### 阶段二：监督微调（SFT）

| 项目 | 详情 |
|------|------|
| **目标** | 使模型学会「何时推理、何时调工具、如何根据工具结果继续推理」 |
| **冻结模块** | 视觉编码器、音频编码器（继承 Qwen3-Omni 预训练权重） |
| **训练模块** | LLM 全参数微调 |
| **损失函数** | 轨迹级监督学习损失（对正例轨迹的每一步做 next-token prediction） |
| **数据类型** | 阶段一合成的成功轨迹（包含推理思考 s_t、动作 a_t、工具返回 o_t 的完整序列） |
| **效果** | Qwen3-Omni-30B: 13.3 → 18.9 Pass@1（+5.6 pp），无效工具使用率 81.1% → 65.3% |

### 阶段三：OmniDPO（细粒度偏好学习）

| 项目 | 详情 |
|------|------|
| **目标** | 在失败轨迹上精确定位第一个错误，进行细粒度纠错训练 |
| **训练模块** | LLM（在 SFT 基础上继续训练） |
| **损失函数** | DPO 偏好学习损失 |
| **纠错模型** | Gemini-3（识别错误位置并生成纠正后的后续） |

**具体流程**：
1. 收集 SFT 后模型的失败轨迹
2. Gemini-3 分析失败轨迹，找出第一个错误——可能是感知错误（看错/听错）、推理错误（逻辑推错）、工具使用错误（搜错关键词/算错）
3. 构造偏好对：正例 = 正确前缀 + Gemini-3 生成的纠正后续；负例 = 正确前缀 + 原始错误后续
4. 用 DPO 损失训练，使模型偏好正确的纠正方案

| **效果** | Qwen3-Omni-30B: 18.9 → 20.8 Pass@1（+1.9 pp），四类错误率全面下降 |
|------|------|

### 错误率变化（Qwen3-Omni-30B）

| 指标 | 基线 | +SFT | +OmniDPO |
|------|------|------|----------|
| 视觉感知错误 | 31.7% | 32.2% | 30.3% |
| 音频感知错误 | 33.9% | 35.8% | 31.9% |
| 无效工具使用 | 81.1% | 65.3% | 59.4% |
| 推理错误 | 79.7% | 68.1% | 64.4% |
| Pass@1 | 13.3 | 18.9 | 20.8 |

### 公开数据集

**基准构建数据来源**：
- **FineVideo**：43K 视频，广泛领域，平均 4 分钟 — https://huggingface.co/datasets/HuggingFaceFV/finevideo
- **LongVideoBench**：~1K 长视频，约 10 分钟
- **LongVideo-Reason**：~1K 长视频，约 10 分钟
- **COCO 2017**：122K 日常场景图像 — https://cocodataset.org

**基准任务**：360 个任务，9 个真实领域（Geography & Travel 19.2%、History & Society 18.6%、Technology 13.6%、Sports 10.3% 等）

## 5. 关键创新

1. **全模态事件图驱动的基准构建**：通过「信号挖掘 → 事件图构建 → 智能体图扩展 → 事件模糊化」的流水线，结构化地合成需要跨模态推理和工具使用的复杂多跳任务，确保逻辑一致且可验证。

2. **OmniDPO 细粒度轨迹纠错**：不同于传统 DPO 对整条轨迹打好坏标签，OmniDPO 精确定位失败轨迹中的第一个错误点，生成局部纠正偏好对，使偏好学习信号更集中、更高效。

3. **主动全模态感知**：模型可按需请求长媒体中特定时间段的视频帧或音频片段，而非被动接受整段下采样输入，类似人类「先扫全局、再聚焦关键片段」的信息获取策略。

4. **后见之明引导的树搜索轨迹合成**：利用已知答案反推正确路径，同时保留失败路径用于 DPO，数据利用效率高，无需人工编写大量训练轨迹。

5. **揭示全模态智能体的关键瓶颈**：实验表明无效工具使用（59.4%）和推理错误（64.4%）是决定性瓶颈，而非感知能力（约 30%）。这一发现重新定义了全模态模型研究的优先级。

---

## 6. 启示与借鉴

1. **工具使用是全模态智能体的核心瓶颈**：OmniGAIA 揭示 Qwen3-Omni 平均每任务仅调用 0.2 次工具，说明当前全模态模型严重缺乏工具使用能力。我们的 Omni 模型设计应从一开始就考虑工具集成推理，而非仅聚焦感知。

2. **OmniDPO 的训练范式值得借鉴**：「定位第一个错误 → 生成纠正后续 → 局部偏好学习」的流程，比整条轨迹级别的 RLHF/DPO 更精细、更高效。我们在训练任何需要多步推理的模型时都可采用此方案。

3. **事件图方法可扩展到数据构建**：全模态事件图不仅可用于基准构建，也可用于训练数据的自动生成——从原始媒体中挖掘结构化知识，合成高质量多跳问答对。

4. **主动感知降低长媒体处理成本**：按需请求特定片段的方案，比均匀下采样更节省上下文、保留更多关键信息，是我们处理长视频/长音频的理想方案。

5. **SFT + DPO 的两阶段提升路径**：SFT 提供主要能力提升（+5.6 pp），DPO 做精细纠错（+1.9 pp），两者互补。我们的训练流程也应遵循「先模仿正确示范、再纠正错误行为」的范式。
