## 2. 模型架构

### 2.1 视觉编码器

| 项目 | 详情 |
|------|------|
| **名称** | 预训练视觉编码器（具体型号论文中通过消融选定） |
| **说明** | 将图像/视频帧编码为视觉嵌入 E_v，支持视频帧序列处理 |

### 2.2 音频编码器

| 项目 | 详情 |
|------|------|
| **名称** | 统一音频编码器（声学+语音共用） |
| **说明** | 将音频波形编码为音频嵌入 E_a，声学（环境声、音乐）和语音共用同一编码器，避免为每种声音单独建编码器 |

### 2.3 LLM 骨干

| 项目 | 详情 |
|------|------|
| **名称** | 基于 Transformer 的 LLM（OmniVinci-9B） |
| **参数量** | ~9B |
| **位置编码** | RoPE（旋转位置编码），与 CRTE 兼容 |

### 2.4 投影层/适配器

| 模块 | 结构 | 说明 |
|------|------|------|
| **OmniAlignNet** | Vision Query Q_v + Audio Query Q_a → 3 层自注意力 → L2 归一化 → 对比损失对齐 | 核心创新模块。将 E_v 和 E_a 映射到共享全模态潜空间，CLIP 式对比损失确保同一视频的画面与声音在向量空间中靠近 |
| **TEG（Temporal Embedding Grouping）** | 按时间块 T_G 分组 + 组内时间戳排序 | 将视觉与音频嵌入按时间段分组，组内按时间先后排列进 token 序列，隐式编码相对时间 |
| **CRTE（Constrained Rotary Time Embedding）** | 周期性绝对时间编码（与 RoPE 兼容） | 在嵌入层注入与时间戳 t 相关的周期性编码，提供绝对时间信息，与 TEG 的相对时间互补 |

### 2.5 语音解码器

OmniVinci 侧重理解端，语音生成依赖外部 TTS 后端（如 Magpie，MOS 4.63、WER 2.7%）。

## 3. 训练阶段详述

### 训练总览

OmniVinci 采用联合训练策略，同时优化 OmniAlignNet 对比对齐损失和 LLM 语言建模损失。训练仅消耗 0.2T token，约为 Qwen2.5-Omni 1.2T 的 1/6。

### 对齐损失

| 项目 | 详情 |
|------|------|
| **损失函数** | L_o-align = (L_v→a + L_a→v) / 2 |
| **公式** | L_v→a = -1/N Σ_i log(exp(s_ii) / Σ_j exp(s_ij))，其中 s_ij = V_i^T A_j |
| **说明** | 对称 CLIP 式对比损失：同一样本的视觉表示 V 与音频表示 A 相似度最大化，不同样本间最小化 |

### 训练配置

| 项目 | 详情 |
|------|------|
| **训练 token 总量** | 0.2T（约为 Qwen2.5-Omni 1.2T 的 1/6） |
| **训练目标** | OmniAlignNet 对比对齐损失 + LLM next-token prediction 联合优化 |
| **数据规模** | 24M 单模态与全模态对话 |

### 数据管线

| 数据类型 | 说明 |
|------|------|
| **单模态对话** | 图文、视频文、音频文独立对话 |
| **全模态对话（隐式）** | 利用现有音视频 QA 数据集 |
| **全模态对话（显式）** | 通过合成管线生成的跨模态对话（需要同时理解视觉+音频才能回答） |

### 数据规模消融

| 训练 token | 效果 |
|------|------|
| 0.05T | 基线 |
| 0.1T | 提升明显 |
| 0.2T | 最优平衡点 |

### 消融实验关键发现

| 消融项 | 影响 |
|------|------|
| 移除 OmniAlignNet | DailyOmni 下降约 5 pp |
| TEG 分组大小 T_G | T_G=5 最优 |
| 移除 CRTE | 视频+音频联合任务下降 2-3 pp |

### 公开数据集

论文提及的评测基准包括：
- **全模态**：DailyOmni、WorldSense
- **音频**：MMAR、LibriSpeech、AIR-Bench、MMAU
- **视觉**：Video-MME、MVBench、LVBench、MMBench、MMStar、MMMU
- **语音输出**：TTS 后端评估（CozyVoice、Bark、StableCodec、Magpie）

### GPU 配置与训练时长

论文未披露具体 GPU 配置和训练时长。作为 NVIDIA 研究团队的工作，推测使用内部大规模 GPU 集群。核心效率优势在于以 0.2T token 超越 1.2T 的 Qwen2.5-Omni。

## 5. 关键创新

1. **OmniAlignNet 全模态对齐网络**：通过 Q_v/Q_a 双 query 将视觉与音频嵌入投影到共享潜空间，经自注意力与 L2 归一化后做对称 CLIP 对比损失，显式学习跨模态语义对齐。移除后 DailyOmni 下降约 5 pp。

2. **TEG（Temporal Embedding Grouping）**：按时间块分组、组内按时间戳排序，将视觉与音频的相对时间对齐编码进序列结构，让 LLM 在读 token 时自然感知时间先后。

3. **CRTE（Constrained Rotary Time Embedding）**：以与 RoPE 兼容的周期性形式注入绝对时间信息，与 TEG 的相对时间互补，增强长视频时间推理能力。

4. **24M 数据管线**：兼顾隐式（现有数据集）和显式（合成全模态对话）两种全模态学习路径，以极少 token（0.2T）实现数据高效训练。

5. **6 倍数据效率**：仅 0.2T 训练 token 即在 DailyOmni（+19.05）、MMAR（+1.7）、Video-MME（+3.9）上超越使用 1.2T token 的 Qwen2.5-Omni，证明架构设计比数据量更重要。

## 6. 启示与借鉴

1. **架构设计 > 数据堆量**：OmniVinci 以 1/6 的训练 token 超越 Qwen2.5-Omni，说明在全模态对齐、时间建模等关键环节做好架构设计，可以大幅减少训练数据需求。我们应优先投入在架构创新上。

2. **显式跨模态对齐至关重要**：OmniAlignNet 的对比学习方案简单有效（对称 CLIP 损失），移除后性能大幅下降。我们的模型应包含显式的视觉-音频对齐模块，而非仅靠 LLM 自行学习跨模态关联。

3. **时间建模的双重方案**：TEG（相对时间）+ CRTE（绝对时间）的组合为视频理解提供了完整的时间建模方案。我们的视频理解模块应同时考虑相对顺序和绝对时刻。

4. **全模态数据合成思路**：24M 数据管线中"显式全模态对话"的合成方法（需要同时理解视觉+音频才能回答），是解决全模态对话数据稀缺的有效途径。

5. **TTS 后端可组合**：OmniVinci 的理解端和语音输出端解耦，语音生成使用外部 TTS（Magpie 达到 MOS 4.63）。这提示我们可以先聚焦理解能力，语音输出作为可插拔模块后续集成。
